# 2. 支付

一般支付有读写操作都频繁的核心接口如下

- 核心支付 单机TPS一般在500左右
- 取消订单 单机TPS一般在1000左右
- 订单查询 

# 3. 交易

一般支付有读写操作都频繁的核心接口如下

- 创建 单机TPS一般在500左右
- 取消订单
- 订单查询 1S内超时

# 4. 风控

风控更多的是查询接口,而且一般是使用多级缓存（本地缓存、分布式缓存）来降低调用链路的时延

- 黑名单查询 RT在100ms以内，一般使用本地缓存
- 灰产查询 RT在100ms以内，一般使用本地缓存
