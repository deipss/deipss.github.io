# 1. 测试基础

- 大量的线上复盘，由技术基本功不扎实导致的bug非常少，大部分bug是因为不熟悉业务而导致。

## 1.1. 功能测试

### 1.1.1. 用例设计原则

- 指导原则：MCCall和Boehm模型，以及ISO/IEC 25010:2011质量模型
- 方法：使用等价类、边界值、判断表等
- 用例架构：从业务用途、功能模块、优先级、前后端等多种维度去划分、归类

## 1.2. 安全测试

- 对外接口，敏感信息的加密，包括（用户ID，手机号，邮箱）
- 接口返回时，不需要敏感数据，不返回
- 认证与鉴权

## 1.3. 兼容性测试

### 1.3.1. 移动端

- app、h5
- top30 top10的机型
- web浏览器的类型

## 1.4. 用例评审

- 是否有遗漏的场景
- 是否有过度的测试用例
- 异常测试、边界值是否覆盖
